# Execalc Runtime Model

This document defines the runtime layers of Execalc and how they interact at decision time.

## Layered Stack
1. Language Layer (LLM): interpretation + expression only
2. Judgment Layer (Kernel): deterministic orchestration
3. Reflex Layer: encoded instincts that fire on triggers
4. Overlay Layer (Carats): situational strategic lenses
5. Knowledge Layer (Nuggets): atomic logic units with tags and polarity
6. Governance Layer (Prime Directive): constraints + evaluation
7. Persistence Layer (Decision Journal): audit envelope + timeline

## Decision-Time Flow
Input → Kernel routes scenario → Reflexes fire → Carats applied → Nuggets assembled → Prime Directive evaluates → Executive report emitted → Journal envelope stored (optional).

## Non-Negotiables
- No prompt may replace Kernel order-of-operations.
- Governance must be enforceable and auditable.
- Tenant isolation is absolute by default.
