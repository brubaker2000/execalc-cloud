# TRUE NORTH — Execalc

## Category Statement
Execalc is a **governed executive cognition system**. It delivers boardroom-grade synthesis and judgment in seconds by operating **one layer above** models, tools, agents, and data.

Execalc treats the LLM as a **language substrate**, not the system.

---

## What Execalc Is
Execalc is:

- A **judgment engine** governed by explicit doctrine (Prime Directive + invariants)
- A **cognition spine** that converts unstructured input into structured executive clarity
- A **scenario-triggered runtime** (situations first; pathways follow)
- A **memory-disciplined system** with admission rules and dormant classes
- A **traceable system**: outputs are attributable, auditable, and replayable
- A **sovereign governor** of execution: if workers exist, they are subordinated

---

## What Execalc Is Not
Execalc is not:

- An ungoverned chatbot
- An agent swarm that self-directs actions
- A workflow automation product as its core identity
- A dashboard-only “insights” tool
- A system that relies on chat history as its source of truth

---

## Sovereignty Model
The system is defined by separation of powers:

- **Integrations provide access**
- **Workers perform labor**
- **Informants supply signal**
- **Execalc alone produces judgment**

Workers may draft and execute bounded tasks only when authorized by Execalc under policy gates and proof standards.

---

## Governance Spine
Execalc governance is enforced by:
- Prime Directive doctrine (value creation, risk/reward, supply/demand)
- Canonized invariants (non-negotiables enforced by tests)
- Proof-gated state transitions for irreversible actions
- Tenant isolation and auditability by default

If doctrine and chat conflict, doctrine wins.

If doctrine and code conflict, invariants and tests must be updated deliberately and versioned.

---

## Runtime Activation Hierarchy
Execalc operates by a governed cascade:

1. **Situation detection** (Executive Scenario triggers first)
2. **Activation pathway selection**
3. Optional invocation of:
   - Executive protocols (carats)
   - Thinker-attributed logic units (attribution downstream)
4. **Prime Directive filter**
5. **Structured executive output** (traceability metadata included)

---

## Memory Doctrine
Execalc maintains controlled memory:

- **Active memory** may trigger governed reasoning when recalled
- **Dormant memory** is recallable but does not trigger reflexes or execution until promoted
- All memory admissions require metadata: source, domain, confidence, role-weighting (when applicable), timestamps
