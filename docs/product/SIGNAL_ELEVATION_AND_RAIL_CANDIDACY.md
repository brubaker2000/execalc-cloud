# Signal Elevation and Rail Candidacy Doctrine

## Purpose

This doctrine defines how live conversational reasoning can be evaluated for elevation into more durable artifacts such as right-rail insights, memory candidates, doctrine lines, and reusable executive nuggets.

It begins from a practical observation:

> Not everything said in the main reasoning stream deserves elevation.  
> But some phrases clearly do.

---

## Core Observation

In conversational outputs, phrases that receive emphasis are often carrying unusual structural weight.

They tend to be:
- distinctions that hold the argument together
- doctrine lines worth reusing
- category-defining claims
- memorable compression of a larger idea
- likely future recall objects

This means visible emphasis is often an early form of artifact detection.

---

## Key Distinction

**Bolding is not the doctrine.**

Bolding is only a presentational clue.

The real system should decide elevation through governed classification, not surface formatting alone.

A phrase can be bold and ephemeral. A phrase can be unformatted and canonical.

Elevation is determined by substance, not styling.

---

## Elevation Ladder

Execalc should recognize several increasing levels of signal elevation:

```
Emphasis-Worthy → Nugget Candidate → Rail Candidate → Memory Candidate → Doctrine Candidate
```

### 1. Emphasis-Worthy
A phrase carries unusual explanatory or rhetorical weight.

### 2. Nugget Candidate
A phrase or sentence expresses a reusable atomic insight.

### 3. Rail Candidate
A phrase is strong enough to stand outside the flow in a sidebar or executive brief surface.

### 4. Memory Candidate
A phrase is material, durable, and decision-relevant enough to be considered for governed memory admission.

### 5. Doctrine Candidate
A phrase is category-defining or operationally foundational enough to warrant formal doctrine consideration.

---

## Candidacy Criteria

A phrase or unit of text becomes a stronger candidate for elevation when it exhibits one or more of the following:

| Criterion | Question |
|---|---|
| **Materiality** | Does it matter to the central argument or decision? |
| **Distinctiveness** | Does it say something non-generic or unusually clarifying? |
| **Compression Value** | Does it condense a larger truth into memorable language? |
| **Reuse Potential** | Could it appear later in doctrine, memory, a deck, a summary, or a product explanation? |
| **Decision Relevance** | Would preserving it improve future executive judgment? |
| **Durability** | Will it likely remain useful beyond the current exchange? |
| **Traceability** | Can it be linked back to a clear context, source, and reasoning chain? |

---

## Artifact Classes

When elevated, a signal should be typed explicitly:

- distinction
- doctrine line
- executive summary line
- heuristic candidate
- memory candidate
- warning or risk line
- opportunity line
- category thesis line
- rail nugget

---

## Rail Candidacy Rule

A phrase is rail-worthy when it can:

1. stand alone
2. preserve meaning outside the surrounding paragraph
3. help orient the operator quickly
4. justify visual elevation through structural importance

**The rail is not for decoration. It is for compressed value.**

---

## Operational Principle

The middle frame contains reasoning in motion. The rail contains the most durable or actionable outputs separating themselves from that motion.

That means signal elevation should function as a governed extraction path from:

```
live reasoning
  → highlighted signal
    → candidate artifact
      → durable memory or doctrine when justified
```

---

## Evaluation Questions

Before elevation, ask:

- Is this the hinge of the argument?
- Is this a phrase worth remembering later?
- Does this clarify a key distinction?
- Would a decision-maker benefit from seeing this outside the flow?
- Is this closer to insight than to support detail?

---

## Relationship to Memory Admission

Signal elevation candidacy is upstream of memory admission — not identical to it.

A phrase can be rail-worthy without meeting the full memory admission threshold.

The flow is:

```
Signal noticed → Elevation ladder assessed → Artifact class assigned
                                           → If memory-worthy: proceed to admission doctrine
                                           → If rail-worthy only: surface, do not persist
                                           → If doctrine-worthy: proceed to canonization protocol
```

---

## Thesis

Bolding is a visual symptom. Signal elevation is the real system.

Execalc should treat emphasized or emphasis-worthy language as candidate extractable value, then govern its promotion according to materiality, distinctiveness, durability, and reuse potential.

The goal is to prevent valuable reasoning from disappearing into scroll — and equally to prevent trivial language from being treated as durable signal.
