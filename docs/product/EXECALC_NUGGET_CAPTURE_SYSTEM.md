# Execalc Nugget Capture System

## Purpose

This document defines the Nugget Capture System for Execalc.

The purpose of this system is to prevent high-value lines, insights, and strategic thoughts from disappearing into chat scroll.

LLM conversations generate an overabundance of good language and useful ideas.
The problem is not scarcity.
The problem is capture.

Execalc must allow the operator to rapidly identify, save, and later curate those moments of insight.

This system turns fleeting conversation into durable strategic material.

---

# Core Problem

In a normal chat workflow:

conversation
→ good line appears
→ conversation continues
→ line disappears into the scroll

This creates strategic loss.

Execalc must solve the problem of ephemeral insight by allowing the operator to capture promising lines immediately and sort them later.

---

# Core Principle

Not every good line should instantly become doctrine.

The correct workflow is:

conversation insight
→ nugget capture
→ Operator Nugget Library
→ later review
→ promotion to heuristic or archive

This prevents doctrine inflation while ensuring that valuable material is not lost.

---

# Trigger Phrase

The primary trigger word is:

Nugget

The operator may also use natural variations such as:

- that's a nugget
- save that nugget
- capture that nugget
- good nugget

The shortest valid command should be:

Nugget.

When this command appears, Execalc should attempt to determine which recent line or paragraph the operator intends to preserve.

---

# Immediate System Goal

The Nugget Capture System must allow the operator to preserve a line in roughly two seconds without interrupting the flow of conversation.
