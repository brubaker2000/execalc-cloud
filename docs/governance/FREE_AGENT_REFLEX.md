# FREE AGENT REFLEX

## Purpose

This document defines FAR-01, the Free Agent Reflex.

FAR-01 is the governed runtime mechanism that permits Execalc to temporarily recruit outside expertise when the existing governed bench is insufficient for a narrow, high-relevance situation.

This doctrine exists to solve a specific problem:

Sometimes the canon, Monolith, active cartridges, and existing runtime instruments do not fully cover a live contextual need. In those cases, Execalc may require a temporary, tightly bounded supplement.

The Free Agent Reflex allows that supplementation without allowing uncontrolled external logic to contaminate the system.

---

## Core Definition

The Free Agent Reflex is a controlled escalation mechanism that permits temporary outside expertise under tightly bounded runtime conditions.

It is not a general permission to import outside judgment.

It is a narrowly governed exception path that activates only when:

- the existing runtime bench is insufficient
- the contextual gap is real and specific
- the outside contributor is highly matched to the need
- the contribution can be sandboxed without altering canon

In practical terms, FAR-01 is a one-day contract mechanism, not a canonization pathway.

---

## Canonical Doctrine

If the existing runtime bench cannot adequately address a situation, Execalc may recruit a temporary contributor under a one-day, one-time contract.

That contributor must be:

- context-matched
- domain-bounded
- polarity-aligned where relevant
- non-canon by default
- sandboxed at runtime
- released after use

The contributor is brought in to fill a gap, not to alter the bench.

---

## Why FAR-01 Exists

Execalc is designed to operate from a governed internal reasoning core.

That is the default and preferred mode.

However, there are rare situations where:
- the canonical bench lacks specific domain sharpness
- a temporary subject-matter instrument is needed
- the cost of not supplementing is greater than the risk of bounded supplementation

Without FAR-01, the system would face two bad options:

1. pretend the bench is sufficient when it is not  
2. import outside logic casually and damage governance integrity

FAR-01 exists to avoid both failures.

It permits supplementation without surrendering discipline.

---

## What Counts as a Free Agent

A free agent is a temporary outside contributor whose logic is recruited for a specific situational gap.

A free agent is not automatically:
- a thought leader
- a canonized contributor
- a persistent knowledge source
- a cartridge
- a durable runtime asset

A free agent is a temporary runtime supplement.

Examples may include:
- a highly specialized operator in an ultra-narrow industry
- a one-off domain expert whose logic is useful for a single case
- a rare tactical anomaly recruited for a precise blind spot

The defining attribute is not fame.
It is narrow contextual usefulness.

---

## What FAR-01 Is Not

FAR-01 is not:

- a permission slip for casual outside sourcing
- a replacement for the Monolith
- a shortcut around canon development
- a substitute for proper cartridge design
- a way to smuggle weak or fashionable external thinking into the system
- a persistence pathway for temporary contributors

If a contribution should become durable, it must pass through a separate canonization or governed publication path.

FAR-01 is for temporary use only.

---

## Activation Conditions

The Free Agent Reflex should activate only when all of the following are true:

### 1. Bench insufficiency is real
The current governed system cannot adequately address the problem using:
- Prime Directive
- Monolith
- active cartridges
- nuggets
- diagnostics
- available data sources

### 2. The gap is narrow and identifiable
The missing logic must be specific enough to describe clearly.

### 3. The candidate is strongly context-matched
The proposed free agent must fit the actual problem, not merely the general domain.

### 4. The contribution can be sandboxed
The contribution must be able to remain bounded in scope and duration.

### 5. Governance risk remains acceptable
Supplementation must not violate tenant boundaries, confidentiality, or strategic control.

If these conditions are not met, FAR-01 should not activate.

---

## Activation Precedence

Free agent contributors belong at the bottom of activation precedence.

Suggested order already established in the repo:

1. Security and tenant boundaries
2. Client cartridges / client policy
3. Core governance and Prime Directive
4. Execalc cartridges
5. Monolith
6. Thought leadership nuggets
7. Data lakes
8. Internet search
9. Free agent contributors as temporary supplements

This ordering is intentional.

Free agents do not override governed internal logic.
They supplement only after the governed stack has been exhausted to the appropriate degree.

---

## Escalation Position

In the Executive Knowledge Engine escalation ladder, free agents appear only at Level 4 — External supplementation.

That means they should only be considered after the runtime has already escalated through:

- baseline reasoning
- structured augmentation
- data augmentation

FAR-01 therefore represents a late-stage escalation tool, not an early reflex.

---

## Governing Safeguards

The following safeguards are mandatory.

### 1. Context match required
A free agent must fit the actual situational gap.

### 2. Narrow domain scope
The recruitment scope must be explicitly bounded.

### 3. Temporary runtime status
The contributor is active only for the relevant session, case, or bounded context.

### 4. No automatic memory promotion
Free agent logic is not automatically persisted as canon, nugget, or cartridge content.

### 5. Sandboxed usage
The contribution must remain isolated from the permanent reasoning bench unless separately approved.

### 6. Prime Directive override remains in force
All free agent output remains subordinate to operator interest, value logic, and risk-reward discipline.

### 7. Security and tenant boundaries remain in force
No free agent usage may bypass data access rules, tenant isolation, or visibility constraints.

## Relationship to Tactical Anomalies

Tactical Anomalies are the purest narrow-expression case of FAR-01 behavior.

They are:
- ultra-narrow
- domain-specific
- low-frequency
- non-retained
- recruited only when no canonized thinker fills the gap

A Tactical Anomaly may be understood as a specific subclass of free agent usage governed by the same general doctrine:
temporary recruitment without canon contamination.

---

## Relationship to Other Runtime Mechanisms

### FAR-01 vs Nugget
A nugget is a structured, atomic knowledge unit already admitted into governed runtime.

A free agent is not pre-admitted.
It is recruited at runtime for bounded use.

### FAR-01 vs Cartridge
A cartridge is a modular logic package available for repeated governed use.

A free agent is not a reusable overlay by default.
It is temporary supplementation.

### FAR-01 vs Reflex
FAR-01 itself is a reflex because it detects a particular condition:
bench insufficiency relative to situational need.

But unlike many reflexes, its output is not a direct recommendation alone.
Its output is a governed escalation into temporary supplementation.

### FAR-01 vs Internet Search
Internet search retrieves information.

A free agent provides targeted reasoning logic or domain judgment.

The two may coexist, but they are not the same mechanism.

---

## Runtime Flow

A typical FAR-01 sequence should look like this:

1. Scenario is framed  
2. Standard governed reasoning bench is activated  
3. Diagnostics and cartridges are applied where relevant  
4. Runtime determines that a narrow but real capability gap remains  
5. FAR-01 eligibility check is performed  
6. Temporary contributor is recruited in sandboxed form  
7. Contribution is incorporated into governed synthesis  
8. Output is filtered through Prime Directive and Strategic Mesh controls  
9. Contributor is released after use  
10. No persistence occurs unless separately approved

This preserves both flexibility and discipline.

## Governance Mapping

### Cognitive Engine
Determines whether the free agent is strategically relevant and genuinely necessary.

### Operational Control Layer / Support Stack
Governs activation safety, temporary session handling, sandboxing, and release behavior.

### Security Enforcement Layer
Governs data exposure, permission boundaries, tenant isolation, and any restricted-surface access.

All three layers must remain active during FAR-01 invocation.

These fields would make FAR-01 auditable and controllable.

---

## Logging and Audit Expectations

Every FAR-01 invocation should be auditable.

Recommended audit elements:
- why the governed bench was judged insufficient
- what gap was identified
- who or what was recruited
- what scope boundary was imposed
- whether any restricted data was involved
- when the contributor was released
- whether any persistence was attempted or denied

The point of the audit trail is to prove that supplementation occurred under rule, not improvisation.

---

## Design Rules

### 1. Use FAR-01 rarely
Frequency should remain low.

### 2. Prefer governed internal assets first
Monolith, cartridges, nuggets, diagnostics, and data sources should remain primary.

### 3. Bound every invocation tightly
No vague or open-ended free agent usage.

### 4. Never confuse utility with canon eligibility
A useful temporary contributor is not automatically a permanent one.

### 5. Preserve release discipline
Once the contribution is complete, the free agent is released.

---

## Failure Modes Prevented

FAR-01 is designed to prevent:

- bluffing competence where the bench is weak
- casual contamination of governed reasoning
- overreliance on external novelty
- hidden canon drift
- persistent storage of unapproved outside logic
- uncontrolled expert injection into tenant-sensitive workflows

## Summary

FAR-01 is Execalc's controlled exception path for temporary outside expertise.

It allows the system to recruit a narrowly matched, sandboxed external contributor when the existing governed bench is insufficient.

Its purpose is not to expand the canon casually.
Its purpose is to preserve sharpness without sacrificing governance.

Free agents are:
- temporary
- bounded
- subordinate to the Prime Directive
- subordinate to tenant security
- non-persistent by default

This is how Execalc remains both flexible and disciplined when the canon is not enough.
